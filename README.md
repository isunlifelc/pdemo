# 产品Demo 搭建工具

一个类似 Figma 的多页面可视化搭建工具，支持快速创建和编辑产品演示页面。

## ✨ 功能特性

### 核心功能
- 🎨 **无限画布** - 自由拖拽、缩放，随意摆放页面
- 📱 **可视化编辑** - 所见即所得的页面编辑器
- 🔄 **实时预览** - 左侧编辑，右侧实时预览效果
- 💾 **持久化存储** - 所有数据自动保存到本地
- 📦 **模块化设计** - 支持多种预设组件卡片

### 页面管理
- ➕ **新建页面** - 一键创建新页面
- ✏️ **编辑页面** - 随时修改已有页面内容
- 📋 **复制页面** - 快速复制生成相似页面
- 🗑️ **删除页面** - 清理不需要的页面
- 📏 **自定义尺寸** - 支持配置页面宽度和高度

### 支持的模块类型
- 💬 用户消息 - 模拟用户输入的对话气泡
- 🤖 AI 文本 - AI 回复的普通文本
- 📝 AI 标题 - AI 回复的标题
- 🏨 酒店卡片 - 完整的酒店信息展示卡片
- ✈️ 机票卡片 - 航班信息展示卡片
- 🚂 火车票卡片 - 火车出行信息
- 🎫 门票卡片 - 景点门票信息
- 🗺️ 地图卡片 - 地图展示
- 💡 追问文本 - 追问选项
- 🔘 建议选项 - 建议胶囊按钮

## 🚀 快速开始

### 方法 1：作为 Qoder Skill 安装（推荐）

在 QoderWork 中输入：
```
/install-skill-dependency
```

然后选择此技能进行安装。

### 方法 2：手动使用

1. 克隆或下载本仓库
2. 在浏览器中打开 `index.html`
3. 建议使用本地服务器：
   ```bash
   # Python
   python -m http.server 8000
   
   # Node.js
   npx http-server -p 8000
   ```
4. 访问 http://localhost:8000/index.html

## 📖 使用指南

### 创建第一个页面

1. **点击"页面"按钮**
   - 位于左侧边栏顶部
   - 会打开编辑器弹窗

2. **编辑页面内容**
   - 从底部模块栏添加组件
   - 拖拽调整模块顺序
   - 编辑每个模块的内容
   - 右侧实时查看预览

3. **保存页面**
   - 点击右上角"保存到画板"按钮
   - 输入页面名称（必填）
   - 配置页面尺寸（默认 375×812px）
   - 点击确定，页面自动添加到画板

### 页面操作

#### 编辑页面
1. 鼠标悬停在页面卡片上
2. 点击顶部的蓝色"编辑"按钮（铅笔图标）
3. 修改内容后保存
4. ✅ 原页面被更新（位置不变）

#### 复制页面
1. 鼠标悬停在页面卡片上
2. 点击顶部的绿色"复制"按钮
3. ✅ 立即在原页面附近创建副本

#### 删除页面
- 点击页面卡片左上角的红色关闭按钮
- 或在左侧边栏点击删除图标

### 画布操作

- **平移** - 在空白处拖拽
- **缩放** - 鼠标滚轮 / 双指滑动 / 工具栏按钮
- **定位页面** - 点击左侧边栏的页面项
- **查找页面** - 使用侧边栏的定位按钮

## 🔧 技术细节

### 数据存储
- 使用 localStorage 持久化存储
- 刷新页面不会丢失数据
- 清除浏览器缓存会重置数据

### 数据结构
```javascript
{
  id: 1,
  type: 'page',
  name: '首页设计',
  file: 'blob:http://...',
  pageData: {
    pageName: '首页设计',
    modules: [...],      // 结构化模块数据
    previewHtml: '<!DOCTYPE html>...',
    displayWidth: 375,
    displayHeight: 812,
    timestamp: 1234567890
  },
  width: 375,
  height: 812,
  x: 100,
  y: 100
}
```

### 通信机制
- 父子页面通过 postMessage 通信
- 使用 localStorage 中转大数据
- 避免 URL 过长问题

## 💡 使用技巧

### 快速批量创建

2. 多次点击"复制"按钮

## ⚠️ 注意事项

1. **使用本地服务器**
   - 由于浏览器安全策略，必须使用本地服务器
   - 推荐：VS Code Live Server、Python http.server、Node.js http-server

2. **数据备份**
   - 定期导出重要页面
   - 可以使用编辑器的"下载页面"功能

3. **浏览器兼容性**
   - 推荐使用最新版 Chrome、Edge、Firefox
   - 需要支持 localStorage 和 postMessage

4. **性能优化**
   - 单个页面建议不超过 20 个模块
   - 过多页面时可以使用删除功能清理

## 🛠️ 开发说明

### 添加新模块类型

1. 在 `editor.html` 中添加模块配置
2. 实现 `renderModuleFields()` 渲染编辑表单
3. 实现 `renderModulePreview()` 渲染预览
4. 在底部模块栏添加对应的按钮

### 修改样式

- 全局样式在 `<style>` 标签中
- 使用 Tailwind CSS 风格的命名
- 保持响应式设计
